This course continues its predecessor with the following three chapters:

Course "Software Architecture" (10 ETCS)

Software architecture is a part of software engineering with covering questions like:
How to describe and rate an architecture? How to specify components and the communication in between? Are there common design patterns? How to classify such patterns? Are there principles to design architectural patterns systematically? Which are the consequences for the architecture when system specifications change? Is an architecture scalable in its performance?

Such questions and more are covered in the following chapters:

  1. Software Architecture: An Introduction
  2. Software Systems and Architectures
  3. Architectural View and Aspects
  4. Architectural Patterns
  5. Programm Frameworks
  6. Architectures for Component Spftware
  7. Description Techniques for Architectures
  8. Designing Software Architectures

Course "Internet Security 1" (5 ETCS)

This course introduces the topic of security in the Internet and is divided into four chapters:

  1. General introduction
    • General protection goals (confidentiality, integrity, authenticity and availability)
    • Specific threats, such as viruses or worms
  2. Encryption
    • Encryption (symmetric, asymmetric) and hash functions
    • Digital signatures and certificates
  3. Security from the perspective of an usual web user
    • Secure email and web surfing
    • Securing remote control
    • Protection systems (firewall, antivirus)
  4. Security from the perspective of a content provider
    • Securing webserver
    • Firewalls
    • Organizational security

The complete course description in German (only)

Dieser Kurs führt in das Thema Sicherheit im Internet ein. Wesentliche Ziele des Kurses sind, dass die Teilnehmer anschließend wissen, welchen Gefährdungen ihr Rechner ausgesetzt ist, welche Schutzmaßnahmen möglich sind. Darüber hinaus sollen die Teilnehmer die Literatur aus diesem Gebiet verstehen und von ihr profitieren können. Der Kurs besteht aus vier Teilen.

Der erste Teil behandelt die grundlegenden Konzepte des Themas Sicherheit. Vertraulichkeit, Integrität, Authentizität und Verfügbarkeit werden als allgemeine Schutzziele vorgestellt. Weiterhin enthält der erste Teil auch Abschnitte zu konkreten Bedrohungen, wie Viren, Würmer und den möglichen Problemen beim Surfen im Netz.

Der zweite Teil behandelt die technischen Grundlagen, mit denen man die Schutzziele erreichen kann. Hierzu gehören symmetrische und asymmetrische Verschlüsselungsverfahren sowie Hash-Funktionen. Es wird vorgestellt, wie man mit Hilfe dieser Verfahren sicherstellen kann, dass Dokumente nicht verändert wurden und wie eine digitale Unterschrift aussieht. Ebenso behandelt die Kurseinheit die Grundlagen von Zertifikaten.

Im dritten Teil wird das Thema Sicherheit aus der Perspektive eines World Wide Web Benutzers beleuchtet. Es wird vorgestellt, wie man sicher durch das Internet surfen kann und wie man sicher per Email kommuniziert. Weiterhin wird darauf eingegangen, wie man sicher an entfernten Rechnern arbeiten kann. Anschließend wird vorgestellt, was man zu Schutz seines privaten Rechners tun kann und soll.

Der vierte Teil beleuchtet das Thema Sicherheit aus der Perspektive eines Anbieters von Internet Inhalten. Es wird vorgestellt, worauf man bei der Konfiguration von Web-Servern achten muss. Daneben wird in diesem Teil auch auf Firewalls, ihre Aufgaben, ihre Architektur, ihre Konfiguration und ihr Betrieb eingegangen. Den Abschluss bildet ein Abschnitt mit organisatorischen Hinweisen zum Thema Sicherheit. Es wird vorgestellt, wie die interne Organisation einer Firma oder Behörde aussehen kann, so dass dem Thema Sicherheit Rechnung getragen wird und es werden Prozesse vorgestellt, mit denen man mehr Sicherheit erreichen kann.

Course "Object Orientated Programming (OOP)" (10 ETCS)

This course covers deeply the fundamental aspects of object orientation. These concepts are taught with the language Smalltalk which defined once the object orientated paradigm and implements them in a plain and origin way. The course covers the following chapters:

  1. Basic concepts of OOP (objects, relations, references, messages, states)
  2. Systematic of OOP (classes, metaclasses, abstract classes, inheritance)
  3. Types in OOP (type definition, equivalence, extension, subtyping and inclusion polymorphism, types and classes, generic types and parametric polymorphism)
  4. Java 6 (objects and types, classes, generic types, interfaces)
  5. Other programming languages (C#, C++, Eiffel)
  6. Problems of OOP (problem of substitutability, fragile-base-class-problem, bad traceability, ...)
  7. Object orientated style (naming, formatting, stiles, Law of Demeter, ...)

This course I already passed successfully in the year 2013. I can say, that was the best and most interesting course I ever took throughout both of my studies.

University of Applied Sciences in Mittweida

In the years from 2005 to 2010 I studied full time in total of nine semester in the field of practical computer science and graduated as Diplom Informatiker (FH).

This study, nowadays turned into Bachelor/Master, can be split up into two parts, the basics and the specialisation. The first three semesters cover the basic sciences like mathematics, physics or electrical technique and electronics. In the following 5 semesters more special and practical topics are covered like software engineering, graphical systems or databases.
In this stage of specialization I focused in the field of software development and engineering.
